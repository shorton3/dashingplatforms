MailboxOwnerHandle * LocalMailbox::createMailbox ( const MailboxAddress &  localAddress)
static

Allows applications to create a mailbox and get a handle to it.

Since we do not want applications to have direct access to a mailbox object this method is provided. Applications that create mailboxes use this method to get a handle to their mailbox rather than have direct access. The handle returned is a owner handle which has all the privileges of the actual mailbox (getMessage/post etc). This method creates the mailbox, activates it (which in turn registers it with the lookup service) and then creates an owner handle, acquires it and then returns that handle. Both the mailbox and the returned handle are created on the heap.

MessageBase * LocalMailbox::getMessageNonBlocking ( )
virtual

Will not block until a message is available.

If the application has a need to do non-blocking calls to get (or check to see if) messages pending on the mailbox queue, then the application needs to implement this call -instead- of calling MailboxProcessor::processMailbox(). If this is indeed the case, then the application will either need to implement the case switching on the Message Type -or- implement the find for the MessageHandlerList (functor lookup).

Note that all non-Proxy Mailbox types use the LocalMailbox's getMessage method to dequeue MsgMgr messages.

Returns
NULL if no message available (and if mailbox deactivated).

void LocalMailbox::startReactor ( void *  arg)
staticprotected

This static method is called in a new thread context to start the ACE Reactor event processing loop during mailbox activation. This will run the event loop until the <ACE_Reactor::handle_events> or <ACE_Reactor::alertable_handle_events> methods returns -1, the <end_reactor_event_loop> method is invoked, or the <ACE_Time_Value> expires.
