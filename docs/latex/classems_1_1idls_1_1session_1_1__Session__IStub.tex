The \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I} interface provides capabilities to manage the client-\/server connection. Its main purpose is to enable either a client or server to detect the loss of communication with the associated party.

For a single communication session between an N\+MS and an E\+MS, there are two \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I} objects. One is maintained on the N\+MS; the other one is maintained on the E\+MS. The \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I} object maintained on the E\+MS is actually an Ems\+Session\+\_\+I, while the \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I} object maintained on the N\+MS is actually an Nms\+Session\+\_\+I (both inherit from \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I}).

Each \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I} object is responsible to \char`\"{}ping\char`\"{} the other \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I} object periodically to detect communication failures. Exactly when this is done is up to the implementation.

When a \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I} object detects a communication failure, or when the end\+Session operation is called on it, all resources allocated with that communication session must be freed and the \doxyref{Session\+\_\+I}{p.}{interfaceems_1_1idls_1_1session_1_1Session__I} object must be deleted.
